# MassMotion 产品设计文档 v2.0

**Slogan:** 精品短剧，一站发现（Curated Short Dramas, Discover with Ease）

**定位转型：** 播放量驱动 → 豆瓣式精品策展 + MyDramaList 式发现

---

## 1. 产品定位

### 1.1 用户画像

**核心用户：**
- 中文短剧爱好者（海外华人 + 国内用户）
- 年龄：18-45 岁
- 痛点：
  - YouTube 上短剧质量参差不齐，找剧困难
  - 缺少专业评分和推荐
  - 想快速发现符合自己口味的精品剧

**使用场景：**
- 🎯 场景 1：晚上想看剧，不知道看什么 → 来 MassMotion 看编辑推荐
- 🎯 场景 2：喜欢霸总甜宠，想找类似的剧 → 按标签筛选
- 🎯 场景 3：看完一部剧，想找同演员的其他剧 → 点击演员查看 filmography

### 1.2 核心价值主张

**For 短剧爱好者**
- ✅ 只推荐精品（策展评分 ≥ 6.0）
- ✅ 专业评分体系（豆瓣 + 策展双评分）
- ✅ 丰富元数据（演员、类型、情绪标签）
- ✅ 免费观看（全部 YouTube 链接）

**vs. 竞品：**
| 平台 | 优势 | 劣势 |
|------|------|------|
| YouTube 搜索 | 内容全 | 质量参差，无评分，无发现 |
| 豆瓣 | 评分权威 | 短剧条目少，无播放链接 |
| MyDramaList | 元数据丰富 | 短剧覆盖少，界面老旧 |
| **MassMotion** | **精品策展 + 丰富元数据 + 免费播放** | - |

## 4. 核心用户流程

### 4.1 新用户首次访问

```
1. 进入首页
   ↓
2. 看到 Hero Banner（编辑精选轮播）
   ↓
3. 浏览"编辑推荐"区域
   ↓
4. 点击感兴趣的剧集 → 进入详情页
   ↓
5. 查看评分、简介、演员
   ↓
6. 点击"在 YouTube 观看" → 跳转 YouTube
   ↓
7. 返回 MassMotion，点击"相似推荐" → 继续发现
```

### 4.2 按情绪找剧

```
1. 首页 → 滚动到"按情绪浏览"
   ↓
2. 点击"🍬 甜蜜剧场" → 查看甜蜜类剧集
   ↓
3. 选择一部剧 → 详情页
   ↓
4. 观看
```

### 4.3 按演员找剧

```
1. 详情页 → 点击演员头像
   ↓
2. 进入演员详情页
   ↓
3. 查看该演员的所有参演作品
   ↓
4. 选择其他剧集 → 观看
```

### 4.4 高级筛选

```
1. 顶部导航 → 点击"浏览"
   ↓
2. 设置筛选条件：
   - 类型：现代都市 + 霸总
   - 情绪：甜蜜
   - 评分：≥ 8.0
   ↓
3. 查看筛选结果（实时更新）
   ↓
4. 选择剧集 → 观看
```

## 5. UI/UX 设计要点

### 5.1 视觉风格

**色彩方案：**
```
主色调：深蓝 (#1a1a2e) - 专业、沉稳
强调色：金色 (#f39c12) - 精品、高级
辅助色：
  - 甜蜜：粉色 (#ff6b9d)
  - 催泪：蓝色 (#4a90e2)
  - 爽感：橙色 (#ff6b35)
  - 搞笑：黄色 (#ffd93d)
  - 狗血：紫色(#a855f7)
```

**字体：**
- 中文：思源黑体（Noto Sans SC）
- 英文：Inter
- 评分数字：Poppins（更现代）

**圆角：**
- 卡片：8px（柔和）
- 按钮：6px
- 标签芯片：16px（全圆角）

### 5.2 评分显示

**双评分系统：**
```
⭐ 策展 8.5  (金色星星 - 编辑团队评分)
💬 豆瓣 8.2  (灰色对话框 - 用户评分)
```

**评分等级颜色：**
- 9.0+ : 金色（#f39c12）- 神剧
- 8.0-8.9 : 绿色（#27ae60）- 精品
- 7.0-7.9 : 蓝色（#3498db）- 优秀
- 6.0-6.9 : 灰色（#95a5a6）- 合格

### 5.3 情绪标签（Mood Chips）

**设计：**
```
┌─────┐
│ 🍬  │  甜蜜  (粉色背景)
└─────┘

┌─────┐
│ 😭  │  催泪  (蓝色背景)
└─────┘

┌─────┐
│ 🔥  │  爽感  (橙色背景)
└─────┘
```

### 5.4 响应式设计

**断点：**
- 手机：< 640px
- 平板：640px - 1024px
- 桌面：> 1024px

**适配策略：**
| 元素 | 手机 | 平板 | 桌面 |
|------|------|------|------|
| 导航 | 底部 Tab Bar | 顶部导航 | 顶部导航 |
| 剧集网格 | 2 列 | 3 列 | 4 列 |
| Hero Banner | 纵向卡片 | 小图轮播 | 大图轮播 |

### 5.5 加载状态

**骨架屏（Skeleton）：**
- 首页加载：显示卡片骨架
- 详情页加载：显示内容骨架
- 避免空白页面

**图片懒加载：**
- 使用 `loading="lazy"`
- 占位符：模糊背景色

### 5.6 无障碍（A11y）

**要求：**
- ✅ 所有图片添加 alt 文本
- ✅ 键盘导航支持（Tab 键）
- ✅ ARIA 标签完整
- ✅ 对比度符合 WCAG AA 标准

## 8. 数据消费示例

### 8.1 读取剧集数据

```typescript
// src/lib/data.ts
import dramas from '@shortdata/dramas.jsonl';

export interface Drama {
  id: string;
  titleChinese: string;
  titleEnglish: string;
  poster: string;
  curatedScore: number;
  doubanRating?: number;
  // ... (参考 DATA_REQUIREMENTS.md)
}

export function getAllDramas(): Drama[] {
  return dramas;
}

export function getDramaById(id: string): Drama | undefined {
  return dramas.find(d => d.id === id);
}

export function getEditorPicks(): Drama[] {
  return dramas.filter(d => d.isEditorPick);
}

export function getHighRated(minScore = 8.0): Drama[] {
  return dramas
    .filter(d => d.curatedScore >= minScore)
    .sort((a, b) => b.curatedScore - a.curatedScore);
}

export function getDramasByMood(mood: string): Drama[] {
  return dramas.filter(d => d.moods.includes(mood as any));
}

export function getDramasByGenre(genre: string): Drama[] {
  return dramas.filter(d => d.genres.includes(genre));
}
```

## 9. 性能优化

**关键指标：**
- ✅ LCP（Largest Contentful Paint）< 2.5s
- ✅ FID（First Input Delay）< 100ms
- ✅ CLS（Cumulative Layout Shift）< 0.1

**优化策略：**
1. **图片优化**
   - WebP 格式（降低 30% 体积）
   - 响应式图片（srcset）
   - 懒加载（viewport 外不加载）

2. **代码分割**
   - 按路由分割（React.lazy）
   - 详情页组件按需加载

3. **数据缓存**
   - Service Worker 缓存 JSON 数据
   - 浏览器缓存策略（1 天）

4. **CDN**
   - 图片托管到 CDN（如 Cloudflare Images）
   - JSON 数据通过 CDN 分发
